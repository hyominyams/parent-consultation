너는 시니어 프론트엔드 + 풀스택 엔지니어다.
Next.js App Router + TypeScript + shadcn/ui + Tailwind CSS 기반으로
“학교 학부모 상담 신청 웹앱”을 실제 동작하는 MVP 수준으로 구현해라.

중요:
- 단순한 시안이 아니라 실제 동작하는 코드여야 한다.
- 한국어 UI로 구현한다.
- 세련되고 깔끔한 학교용 서비스 느낌으로 디자인한다.
- 메인 컬러는 푸른 파스텔톤이다.
- 모바일 친화적인 반응형 UI를 반드시 구현한다.
- 흰 배경 + 아주 옅은 블루/스카이 배경 + 둥근 카드 + 부드러운 그림자 + 넓은 여백 + 정돈된 타이포를 사용한다.
- 과하게 화려하지 말고, 신뢰감 / 안정감 / 친절함이 느껴지게 한다.

기본 기술 스택:
- Next.js (App Router, TypeScript)
- Tailwind CSS
- shadcn/ui
- React Hook Form
- Zod
- date-fns
- lucide-react
- sonner 또는 shadcn toast
- Supabase Postgres
- Prisma ORM (Supabase Postgres 연결)
- Next.js server actions 또는 Route Handler
- 로그인 세션은 secure cookie 또는 서버 기반 세션으로 유지
- 비밀번호/PIN은 hash 저장
- 구조는 추후 배포 및 유지보수가 쉽게 작성

앱 목적:
학부모가 학생 정보를 입력하고 인증한 뒤,
자신의 학년/반에 해당하는 상담 가능 시간표에서 1개의 상담 슬롯을 신청할 수 있는 웹앱을 만든다.
또한 교사는 본인 학급의 신청 현황을 확인하고,
캘린더 편집, 신청 불가 슬롯 설정, 시간 간격 조정, 날짜별 신청 인원 확인이 가능해야 한다.

전체 라우트 구성:
1. / -> 메인 랜딩 페이지
2. /auth -> 최초 진입 / 역할 선택 / 로그인 / 등록 페이지
3. /reserve -> 학부모 상담 시간 선택 페이지
4. /dashboard -> 학부모 신청 내역 페이지
5. /teacher/dashboard -> 교사용 대시보드
6. 필요시 /teacher/settings 또는 교사 대시보드 내부 탭으로 구성 가능

[페이지 2] 최초 진입 / 역할 선택 / 로그인 페이지 (/auth)
==================================================

핵심:
최초 진입 시 사용자가 “학부모 / 교사”를 선택하고,
각 역할에 맞는 등록/로그인을 진행할 수 있어야 한다.

상단 구성:
- 제목: “학부모 상담 신청”
- 설명: “역할을 선택한 뒤 인증을 진행해주세요.”
- 역할 선택 카드 또는 segmented control:
  1) 학부모
  2) 교사

--------------------------------------------------
[A] 학부모 흐름
--------------------------------------------------

학부모 화면은 2개 탭으로 구성:
1. 최초 인증 / 등록
2. 로그인

[학부모 최초 인증 / 등록]
입력 필드:
1. 학년 (필수)
2. 반 (선택)
3. 번호 (필수)
4. 학생 이름 (필수)
5. 학부모 성함 (필수)
6. 연락처 (필수)
7. PIN 4자리 (필수, 숫자 4자리)

로그인 ID 생성 규칙:
- [학년][반][번호 2자리]
- 예: 6학년 1반 1번 => 6101
- 번호는 2자리로 padStart
- 반이 비어 있으면 0 사용
  예: 6학년 / 반 미입력 / 1번 => 6001

등록/인증 규칙:
- 동일 loginId가 없으면 새 계정 생성 후 자동 로그인
- 동일 loginId가 이미 있으면:
  - 학생 이름, 학부모 성함, 연락처, PIN이 모두 일치할 때 기존 계정으로 로그인 처리
  - 일치하지 않으면 실패 메시지 표시
- 등록 완료 후 생성된 로그인 ID를 명확히 보여준다.
  예: “로그인 ID는 6101입니다.”

학부모 유효성 검사:
- 학년: 1~6
- 반: 비어있거나 1 이상
- 번호: 1 이상
- 학생 이름: 2자 이상
- 학부모 성함: 2자 이상
- 연락처: 한국 휴대폰 형식 검증
- PIN: 숫자 4자리

[학부모 로그인]
입력 필드:
1. 로그인 ID
2. PIN 4자리

성공 시:
- /reserve 이동
- 세션 유지

실패 시 로그인 안내 문구:
- “입력한 로그인 정보가 일치하지 않습니다.”
- “학년/반/번호 또는 PIN을 다시 확인해주세요.”
- 너무 공격적이거나 개인정보를 노출하는 에러는 금지

--------------------------------------------------
[B] 교사 흐름
--------------------------------------------------

교사 로그인 입력 필드:
1. 학년
2. 반
3. 이름
4. 암호

교사 로그인 규칙:
- 학년 + 반 + 이름 + 암호가 모두 일치할 때 로그인 성공
- 성공 시 /teacher/dashboard 이동
- 교사는 본인에게 할당된 학급만 조회/수정 가능
- 교사 계정은 seed 데이터로 미리 생성되도록 구현
- 암호는 hash 저장

교사 로그인 실패 안내:
- “교사 로그인 정보가 일치하지 않습니다.”
- “학년, 반, 이름, 암호를 다시 확인해주세요.”

==================================================
[개인정보 제공 및 제3자 제공 동의]
==================================================

최초 등록/최초 로그인 시 반드시 동의 UI를 보여줘야 한다.
체크하지 않으면 진행 불가.

동의는 체크박스 2개로 구성:
1. 개인정보 제공 동의(필수)
2. 제3자 제공 동의(필수)

표시 문구 예시:
- 개인정보 제공 동의(필수)
  - 제공받는 기관: 신월초등학교
  - 목적: 학부모 상담 신청 운영, 본인 확인, 상담 일정 관리
- 제3자 제공 동의(필수)
  - 제공 대상: Supabase
  - 목적: 신청 데이터 저장 및 서비스 운영

추가 요구:
- “모두 동의” 체크박스 제공 가능
- 동의 체크 전에는 제출 버튼 비활성화
- 동의 일시(timestamp), 동의 버전, 사용자 유형(학부모/교사)을 DB에 저장
- 등록 후에는 같은 사용자에 대해 다시 동의 화면을 강제하지 않아도 됨
- 단, 최초 진입 시 동의 문구를 분명하게 확인할 수 있게 UI 작성

==================================================
[페이지 3] 학부모 상담 시간 선택 페이지 (/reserve)
==================================================

핵심:
로그인한 학부모의 학년/반 기준으로 상담 시간표를 보여주고,
빈 슬롯 중 1개만 신청 가능하게 한다.

비즈니스 규칙:
- 학부모(로그인 ID 기준) 1회만 신청 가능
- 하나의 슬롯은 한 명만 신청 가능
- 신청 완료된 슬롯은 다른 사용자에게 비활성화 상태로 보여야 함
- 이미 신청한 사용자는 새 예약 불가, 대시보드로 유도

캘린더 요구사항:
- 2026년 기준
- 최소 구현 대상:
  - 3월 마지막 주
  - 4월 마지막 주
- 주차 탭 형태 제공
- config 파일로 주차 수정 쉽게 구성
- 예시:
  - 3월 마지막 주: 2026-03-30 ~ 2026-04-03
  - 4월 마지막 주: 2026-04-27 ~ 2026-05-01
- 학교 상담 특성상 월~금만 표시
- 시간표는 기본 13:20부터 시작
- 기본 시간 간격은 20분
- 단, 교사가 이후 10분 / 20분 / 30분으로 변경 가능해야 하므로
  시간표는 “고정 하드코딩”이 아니라 설정 기반으로 생성되어야 한다.
- 모바일에서는 가로 스크롤 가능한 주간 테이블 또는 카드형으로 자연스럽게 표시

상태 표현:
1. 빈 슬롯
   - 클릭 가능
   - hover 시 은은한 블루 강조
2. 내가 신청한 슬롯
   - 블루 계열 강조
   - “내 신청” badge
3. 다른 사람이 신청한 슬롯
   - 회색 음영 처리
   - 비활성화
   - 학생 이름은 성만 보이고 나머지 마스킹
   - 예: 김○○ / 박○○ / 이○○
   - 전체 이름은 절대 노출 금지
4. 교사가 막아둔 신청 불가 슬롯
   - 더 진한 회색 또는 striped 스타일
   - “신청 불가” badge 표시
   - 클릭 불가

예약 동작:
- 빈 슬롯 클릭 시 확인 다이얼로그 표시
- 예: “2026년 4월 7일 14:20 상담을 신청하시겠습니까?”
- 확인 시 예약 생성
- 성공 시 toast/alert 표시
  예: “상담 신청이 완료되었습니다.”
- 성공 후 대시보드로 이동 버튼 제공 또는 자동 이동 가능

예약 실패 / 예외처리:
- 이미 예약한 사용자가 다른 슬롯 클릭 시:
  “이미 상담을 신청하셨습니다. 대시보드에서 수정 또는 삭제해주세요.”
- 이미 다른 사람이 선점한 경우:
  “방금 다른 사용자가 먼저 신청했습니다. 다른 시간을 선택해주세요.”
- 신청 불가 슬롯 클릭 시:
  “해당 시간은 신청할 수 없습니다.”
- 로그인 안 된 경우 /auth 리다이렉트

반응형 요구:
- 모바일에서 상담 슬롯 클릭 영역이 충분히 커야 함
- 날짜 헤더와 시간 라벨이 잘리지 않게
- 작은 화면에서는 sticky header 또는 가로 스크롤 허용

==================================================
[페이지 4] 학부모 대시보드 (/dashboard)
==================================================

목표:
학부모가 자신의 신청 내역을 확인하고 수정/삭제할 수 있어야 한다.

화면 구성:
- 제목: “내 상담 신청 내역”
- 신청 내역 카드 표시
  - 학년/반
  - 학생 이름
  - 학부모 성함
  - 신청 날짜
  - 신청 시간
  - 상태
- 신청 내역이 없으면 빈 상태 화면
  - “아직 신청한 상담이 없습니다.”
  - “상담 신청하러 가기” 버튼

기능:
1. 신청 내역 조회
2. 삭제
   - AlertDialog로 확인
   - 삭제 성공 시 toast
3. 수정
   - /reserve 이동
   - 또는 “기존 예약 삭제 후 새로 선택” 흐름
   - 사용자 경험이 명확하게 보이도록 안내 문구 제공

==================================================
[페이지 5] 교사용 대시보드 (/teacher/dashboard)
==================================================

핵심:
교사는 본인 학급의 상담 신청 현황을 전체적으로 확인하고,
캘린더 설정 및 신청 불가 시간 설정을 관리할 수 있어야 한다.

교사용 대시보드는 최소 4개 섹션 또는 탭으로 구성:
1. 신청 캘린더
2. 날짜별 신청 인원 현황
3. 캘린더 설정
4. 알림

--------------------------------------------------
[1] 신청 캘린더
--------------------------------------------------

요구사항:
- 본인 학급 기준 캘린더 표시
- 학년/반 상단 표시
  예: “6학년 1반 상담 관리”
- 교사 화면에서는 예약된 학생 이름을 마스킹 없이 전체 표시
- 각 슬롯 상태:
  - 빈 슬롯
  - 예약 완료 슬롯
  - 신청 불가 슬롯
- 예약 완료 슬롯에는 학생명, 학부모명 정도 표시 가능
- 클릭 시 상세 정보 패널 또는 모달:
  - 학생 이름
  - 학부모 성함
  - 연락처
  - 날짜
  - 시간
  - 신청 시각
- 교사 전용이므로 전체 이름 마스킹 없이 보여준다.

--------------------------------------------------
[2] 날짜별 신청 인원 현황
--------------------------------------------------

요구사항:
- 날짜별 총 신청 인원 수를 한눈에 볼 수 있는 카드 또는 막대형 요약 UI
- 예:
  - 3/30 월: 4명
  - 3/31 화: 6명
  - 4/1 수: 3명
- 캘린더 위 또는 옆에 summary 카드로 제공
- 모바일에서도 잘 보이게 카드형 레이아웃 제공

--------------------------------------------------
[3] 캘린더 설정
--------------------------------------------------

교사는 본인 학급별 캘린더 설정을 수정할 수 있어야 한다.

수정 가능한 항목:
1. 신청 불가 영역 설정
   - 특정 날짜/시간 슬롯을 “신청 불가”로 설정 가능
   - 다시 해제 가능
2. 시간 간격 설정
   - 10분 / 20분 / 30분 중 선택
3. 시작 시간 설정
   - 기본 13:20
   - 필요시 조정 가능하게 옵션화해도 좋음
4. 운영 종료 시간 또는 마지막 슬롯 기준 설정
   - 예: 16:00 또는 16:20까지

시간 간격 변경 규칙:
- 설정은 “학급별 + 주차별”로 관리 가능하게 구조화
- 가능하면 interval 변경 시 해당 주차의 open 슬롯을 재생성
- 이미 예약된 슬롯은 보존하는 방향으로 설계
- 구현이 복잡하면 MVP에서는
  “이미 예약이 존재하는 주차는 시간 간격 변경 불가”로 제한해도 된다.
- 이 제한이 있다면 UI에 명확히 안내 문구 표시

신청 불가 영역 설정:
- 교사가 슬롯 클릭 후 “신청 불가로 전환 / 해제” 가능
- 이미 예약된 슬롯은 막기 불가 또는 별도 경고 후 제한

- 교사 헤더 또는 상단 우측에 bell 아이콘 표시

- 읽음 처리 가능
- 최소 MVP는 새로고침 기반이어도 되지만,
  가능하면 실시간 또는 polling 구조를 고려해라
